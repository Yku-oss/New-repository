### MyBatis
## 概念
````
MyBatis是一个Java持久层的一个框架，是一个能让Java代码里的sql语句连接数据库操作，能极大提高开发效率
````

## 作用
````
可以将sql语句和java代码进行“解耦”，能直接连接数据库，可以专注的写sql本身，Mybatis会自动将数据打包成一个对象放入到数据库中
再将结果映射成对象返回给java代码输出。
````

## 框架理解
````
Mybatis 本身就是一个半自动的ORM(对象关系)框架

sql语句必须手写，但Mybatis会负责将sql的输入参数自动映射成java对象，并把查询结果自动映射成回java对象

半自动的好处就是后面可以进行对sql语句的调优。
````

# 工作流程
````
      定义一个Mapper（数据访问层）接口，里面定义了数据操作的方法
                        👇
      写好一个xml文件的sql语句，并于Mapper接口连接
                        👇
      Mybatis会自动将调用的操作方法与sql语句进行配对，
      然后翻译xml文件中的sql语句，并自动将结果映射成java对象返回
````

## Application 引导类(启动类)
```
 这是每一个类别的启动项，他是每一个框架的启动项。

 当执行这个引导类时，框架工具就会扫描这个类以及附属类的所有注释，然后创建好对象bean
```

## API（Application Programming Interface）
```
Api 是一个应用程序的编程接口：相当于当你调用这个方法时，能看到这个入口。
在代码中遇到很多注释，比如：@Getmapping（）就是一个API接口，包含了路径和请求方法。
```

## Spring MVC
```
MVC = Model（模型） + View（视图） + Conttoller（控制器）
Spring MVC  就是一个专门处理Web请求的一个模块，就是联通前端与后端的信息通道
    工作流程就是：
                         接受HTTP的请求
                              👇
                  找到对应的Controller方法
                              👇
                        返回给客户端（前端）

其包括：
DispatcherServlet（中央调度器）——所有请求先到这里

HandlerMapping（路由映射）——找到对应Controller

Controller（控制器）——@RestController 

HttpMessageConverter（消息转换器）——对象转JSON


启动时就会扫描这个类的所有处理HTTP的注释，并且调用其方法。
```
